Introducing *.carsobj

The java lab8 serialization format

The *.casrobj (car $ object) is the newline terminated serialization format. Annoyed at how json is too readable and simple? Fear not the new notation is here. The format is completely language and data agnoctic and it was designed with no standardization in mind.

It consinst from 3 main data points:
  1. Fields: the simple data that you cary.
    The field notation is as follows field:= name '$' value '\n'
  2. Objects - the collection of fields or other objects. Used to represent complex data.
    The object notation is: object := name '\n' (object|field|collection)* '\n'
  3. Collection - the collection of objects. Each collection is prefixed with the number of values in it for simplicity, and additionally by name, should you require one.
    The array notation is: collection := number '\n' (name '\n')? '[' (object)* ']' '\n'

These are universal data structures. Virtually all modern programming languages support them in one form or another. It makes sense that a data format that is interchangeable with programming languages also be based on these structures.

Examples:
Simple object: CarIdentity Manufacturer $Kia Model $Sportage Year $2015
More complext object: SuperCar CarIdentity Manufacturer $Pagani Model $Huayra Year $2012 Engine Fuel Consumption $12.2 Horse Power $730 Price $1375500 Max Speed $370 Seats $2 Automatic $true Turbo Boosted $true Special Edition $true
Collection: 1 Cars [ SuperCar CarIdentity Manufacturer $Pagani Model $Huayra Year $2012 Engine Fuel Consumption $12.2 Horse Power $730 Price $1375500 Max Speed $370 Seats $2 Automatic $true Turbo Boosted $true Special Edition $true ]
